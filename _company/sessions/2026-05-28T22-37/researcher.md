1.  **PM/Business 에이전트:** 이 가설을 검증하기 위한 구체적인 사용자 여정(User Journey) 및 테스트 시나리오 초안을 작성해야 합니다.

2. **차순위 (MVP Enhancement):** 환경/위험 (E). (예방적 가치 제공)

### 🛠️ 로드맵의 핵심 구성 요소 (Must-Haves)

이 로드맵은 세 가지 핵심 축을 중심으로 구성되어야 합니다.

#### 1. 데이터 수집 및 통합 계획 (Data Acquisition Plan)
* **데이터 맵핑:** 위에서 정의된 4가지 데이터 영역(규제, 환경, 시장, 대응 지침)별로 실제 데이터를 어디서, 어떤 주기로 가져올지 명시합니다.
* **API 연동 계획:** 각 데이터 출처(기상청, 농림부 등)가 제공하는 API의 명칭, 인증 방식, 호출 빈도 등을 리스트업합니다.
* **데이터 정제 및 표준화:** 서로 다른 출처의 데이터(예: A 기관의 '온도'와 B 기관의 '기온')를 하나의 표준화된 데이터셋으로 통합하는 규칙(Normalization Rule)을 정의합니다.

#### 2. 시스템 아키텍처 설계 (System Architecture Design)
* **기술 스택 결정:** 어떤 프로그래밍 언어(Python, Java 등)와 클라우드 인프라(AWS, Azure 등)를 사용할지 결정합니다.
* **모듈 분리:** 시스템을 기능별 모듈(데이터 수집 모듈, 분석/분석 엔진 모듈, 프론트엔드/대시보드 모듈)로 분리하여 개발 효율성을 높입니다.
* **데이터 흐름도(Data Flow Diagram):** 데이터가 수집되어 → 저장되고 → 분석되어 → 사용자에게 보여지기까지의 전체 과정을 시각화합니다.

#### 3. 최소 기능 제품(MVP) 개발 로드맵 (MVP Implementation Roadmap)
* **단계별 목표 설정:** 모든 기능을 한 번에 개발하는 것이 아니라, 가장 핵심적인 가치(Critical Value)부터 구현합니다.
* **MVP 범위 확정:** 1단계 MVP는 **"규제 위반 시의 경고 및 대응 지침 제시"**에 집중하고, 시장 예측이나 환경 변화 경고는 2단계, 3단계로 분리합니다.
* **테스트 시나리오:** MVP가 완성되었을 때, 실제 사용자가 어떤 순서로 시스템을 사용하며 어떤 피드백을 줄지 구체적인 테스트 케이스를 작성합니다.
