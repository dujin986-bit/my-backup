# 📈 [1차 리서치 보고서] 축산 농가 대상 데이터 기반 리스크 관리 시장 기회 분석

**작성 목적:** 현재 축산 시장에서 가장 시급하고 데이터 기반으로 접근 가능한 리스크 관리 영역을 정의하고, 이를 MVP로 구현했을 때의 시장 매력도 및 경쟁 우위를 입증한다.

**핵심 가설 (Hypothesis):** 축산 농가들은 단순히 '정보'를 얻는 것보다, **'데이터 기반의 사후 대응이 아닌 사전 예방적 위험 점수 및 구체적인 행동 지침(Actionable Manual)'**을 제공받을 때 가장 높은 가치를 느낄 것이다.

## 1. 시장 Pain Point 및 잠재적 관심도 분석 (Market Gap Analysis)

| Pain Point 영역 | 축산 농가의 실제 어려움 (Pain Point) | 데이터 기반의 해결 필요성 (Gap) | 시장 관심도/검색 트렌드 (추정) |
| :--- | :--- | :--- | :--- |
| **질병 및 사육** | 질병 발생 시 원인 분석 및 확산 예측이 어렵고, 최신 방역 정보 접근성이 낮음. | **[사전 예방적 리스크 점수화]** - 지역별, 질병별 감염 위험도 예측 및 이동 제한 권고. | **매우 높음** (정책 변화 민감도 반영) |
| **시장 가격 및 유통** | 경락시장 가격 변동성이 크고, 등급별/지역별 실시간 시세 파악이 어려움. | **[종합 시세 예측 모델]** - 기후/사육 데이터 연동을 통한 미래 예상 가격 및 적정 출하 시기 제시. | **높음** (직접적인 수익과 연결) |
| **환경/규제 변화** | 기후 변화나 정부 규제 변화에 따른 사육 방식 변경 및 리스크 관리가 복잡함. | **[규제 변화 영향 분석]** - 특정 규제(예: 환경 규제)가 농가 운영 비용 및 생산성에 미치는 **정량적 영향 분석**. | **중상** (정보의 권위성이 중요) |

**🔎 리서치 기반 결론:** 가장 강력한 진입점은 **'질병/환경 리스크'**와 **'시장 예측'**의 교차점입니다. 즉, **"환경 및 질병 리스크가 축산물의 최종 시장 가치에 미치는 영향을 예측하고, 이를 최소화하는 행동 매뉴얼을 제공하는 것"**이 가장 높은 가치를 지닙니다.

## 2. 경쟁사 벤치마킹 및 차별화 포인트 (Competitor Analysis)

| 경쟁사/모델 | 핵심 기능 및 강점 | 백프로가 차별화할 수 있는 지점 (Opportunity) |
| :--- | :--- | :--- |
| **키우소 어플** | 사용자 간 정보 공유, 커뮤니티 기반의 실질적 경험 공유. (네트워크 효과) | **[정보의 권위성]** : 개인 경험 공유를 넘어, **공신력 있는 공공/학술 데이터를 기반으로 '객관적인 점수'**를 제시해야 함. |
| **일반 축산 관련 커뮤니티** | 광범위한 정보 교류, 최신 정책 공유. (정보의 양) | **[데이터 연동 및 행동 매뉴얼]** : 단순 정보 나열이 아닌, **'위험 점수 → 원인 분석 → 구체적인 조치(Action)'**의 3단계 프로세스 제공. |
| **(가상) 정부/공공 플랫폼** | 가장 신뢰도 높은 공식 데이터 제공. (신뢰도) | **[데이터 종합 및 시각화]** : 여러 공공 API(질병, 기후, 시장)를 **하나의 대시보드에서 통합하고, 사용자가 이해하기 쉬운 '종합 지표'로 변환**하는 능력. |

## 3. MVP Scope 및 데이터 확보 전략 (Action Plan)

**[🎯 MVP 핵심 범위 정의]**

*   **서비스명 (가칭):** 백프로 Risk Score (베타)
*   **핵심 가치:** 농가별 '종합 위험 점수' 및 '주간 행동 매뉴얼' 제공.
*   **사용자 시나리오:** 농가가 로그인 → 지역/가축 정보 입력 → 종합 위험 점수(예: 75점, '주의' 등급) 확인 → '경제적 리스크(예: 출하 시기 조정)' 및 '질병 리스크(예: A 백신 접종 필요)'에 대한 구체적 조치 사항 확인.

**[🛠️ 필수 데이터 셋 및 확보 전략]**

| 데이터 셋 (Data Set) | 데이터 출처 (Source) | 확보 전략 (Acquisition Strategy) | 백프로의 역할 |
| :--- | :--- | :--- | :--- |
| **1. 공공 보건/질병 정보** | 축산물품질평가원, 질병관리청 API (가정) | **API 연동 (우선순위 1)**: API 제공 여부 확인 후, 데이터 수집 파이프라인 구축. | 위험도 지수 산출 및 시각화. |
| **2. 기후/환경 데이터** | 기상청 API (우선순위 2): 지역별 강수량, 평균 기온 변화 추이. | API 연동 및 데이터 정제. | 사육 환경 리스크 점수 산출. |
| **3. 시장/가격 데이터** | 농림축산식품부 데이터 또는 민간 API (우선순위 3): 품목별 최신 도매가 및 추이. | 데이터 크롤링 또는 제휴를 통한 주기적 데이터 수집. | 경제성 리스크 점수 산출. |
