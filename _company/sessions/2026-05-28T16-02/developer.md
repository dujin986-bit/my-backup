### 🚀 1단계: 기술 스택(Tech Stack) 확정 및 의존성 정의

먼저, 프로젝트의 성격(데이터 수집, API 통신, 모델 구동 등)을 고려하여 사용할 핵심 라이브러리 목록을 확정해야 합니다.

**✅ 산출물:** `requirements.txt` (또는 `pyproject.toml`)

**🛠️ 실행 계획:**
1.  **핵심 라이브러리 정의:** 프로젝트의 목적에 맞는 라이브러리(예: `requests` (HTTP 통신), `pandas` (데이터 처리), `scikit-learn` (ML 모델), `python-dotenv` (환경 변수 관리))를 목록화합니다.
2.  **의존성 명시:** 각 라이브러리의 최소/권장 버전을 명시하여 재현 가능한 환경을 구축합니다.

**💡 예시 (가정):**
만약 데이터 수집 및 분석이 주 목적이라면, 다음과 같이 정의합니다.
```text
# requirements.txt
requests>=2.28.1
pandas>=1.4.0
numpy>=1.20.0
python-dotenv>=1.0.0
```

### ⚙️ 2단계: 개발 환경 구축 및 격리 (Virtual Environment)

프로젝트 의존성이 시스템 전역에 영향을 주는 것을 막기 위해 가상 환경(Virtual Environment)을 사용하는 것이 필수적입니다.

**✅ 산출물:** `venv` 디렉토리 구조 및 실행 스크립트 (`setup_env.sh` 또는 `setup_env.bat`)

**🛠️ 실행 계획:**
1.  **가상 환경 생성:** `python -m venv venv` 명령어를 통해 격리된 환경을 만듭니다.
2.  **환경 활성화 스크립트 작성:** 사용자가 쉽게 환경을 활성화할 수 있도록 스크립트를 작성합니다.

**📜 예시 (Shell Script):**
```bash
#!/bin/bash
echo "========================================"
echo "🚀 가상 개발 환경 활성화 시작"
echo "========================================"
python3 -m venv venv
echo "✅ 가상 환경 'venv' 생성 완료."

echo "💡 다음 명령어로 환경을 활성화하세요:"
echo "    source venv/bin/activate"
echo "========================================"
```

### 🔒 3단계: 민감 정보 관리 및 로딩 로직 구현

API 키, 데이터베이스 비밀번호 등 민감한 정보는 코드에 하드코딩해서는 안 됩니다. `.env` 파일을 사용하여 관리하고, 애플리케이션 시작 시 로드하는 로직을 구현해야 합니다.

**✅ 산출물:** `.env.example` 및 환경 변수 로딩 코드 (Python)

**🛠️ 실행 계획:**
1.  **샘플 환경 파일 생성:** `.env.example` 파일을 만들어 어떤 변수가 필요한지 구조만 노출합니다. (실제 키는 제외)
2.  **로딩 로직 구현:** `python-dotenv` 라이브러리를 사용하여 `.env` 파일에서 환경 변수를 읽어와 애플리케이션의 설정 객체에 주입하는 코드를 작성합니다.

**🐍 예시 (Python Code Snippet):**
```python
# config_loader.py
import os
from dotenv import load_dotenv

def load_environment():
    """프로젝트의 환경 변수들을 로드합니다."""
    load_dotenv()
    
    api_key = os.getenv("MY_API_KEY")
    db_url = os.getenv("DATABASE_URL")
    
    if not api_key or not db_url:
        raise ValueError("필수 환경 변수 (MY_API_KEY, DATABASE_URL)가 설정되지 않았습니다. .env 파일을 확인하세요.")
        
    print("✅ 환경 변수 로드 성공.")
    return {
        "API_KEY": api_key,
        "DB_URL": db_url
    }

if __name__ == "__main__":
    try:
        config = load_environment()
        print(f"API Key (일부): {config['API_KEY'][:5]}...")
    except ValueError as e:
        print(f"❌ 설정 오류: {e}")
```
