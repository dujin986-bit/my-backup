# 📊 축산 데이터 기반 위험 관리 시스템 구축을 위한 리서치 보고서

## Ⅰ. 시장 트렌드 재검토: '예측'의 필요성 증대 (Pain Point Validation)

현재 축산업계는 단순한 정보 습득을 넘어, **'미래 위험에 대한 예측과 선제적 대응'**을 가장 큰 니즈로 인식하고 있습니다.

| 트렌드 키워드 | 시장 변화 및 Pain Point | 위험 지수 시스템과의 연결 고리 |
| :--- | :--- | :--- |
| **기후 변화 적응** | 국지적 기상 이변(가뭄, 홍수) 증가로 사료 작물 생산량 예측 불가능. 사육 환경의 변동성이 커짐. | **[환경 위험 지수]** 기상 예측 데이터(강수량, 온도)와 사료 가격 변동성을 결합하여 사육 비용의 급격한 증가 위험 예측. |
| **질병 및 방역 강화** | 인수공통감염병 및 신종 질병 출현 위험 증가. 지역별 질병 유행 패턴 분석의 필요성이 극대화. | **[생물학적 위험 지수]** 공공 보건/질병 통계 API와 지역별 사육 밀도 데이터를 결합하여, 질병 발생 확률이 높은 '핫스팟' 예측. |
| **소비 트렌드 변화** | 소비자들이 '안전성'과 '지속 가능성(ESG)'에 대한 요구가 높아지면서, 생산 과정의 투명성 요구 증대. | **[시장 가치 위험 지수]** 경매장 시세 외에, 공신력 있는 사료/혈통/사육 이력을 통합하여 '최적의 가치'를 예측하고 제시. |

**💡 결론:** 현재 시장은 '무엇이 일어났는지(Descriptive)'보다 **'무엇이 일어날 것 같은지(Predictive)'**에 대한 정보를 가장 필요로 합니다. 이는 위험 지수 예측 시스템의 존재 이유를 명확히 합니다.

## Ⅱ. 경쟁사 분석 및 차별화 포인트 재확인

현재 시장의 주요 플레이어들은 대부분 **'거래'**나 **'정보 나열'**에 집중되어 있습니다.

| 경쟁사/기존 플랫폼 | 주요 기능 | 한계점 (Pain Point) | 백프로의 차별화 전략 |
| :--- | :--- | :--- | :--- |
| **경매장/장터 앱** | 실시간 경락가, 거래 정보 제공 | **[미래 예측 부재]** 현재의 가격과 양만 보여줄 뿐, 미래의 시장 변동성(예: 기후로 인한 사료가 폭등 시)을 반영하지 못함. | **[위험 시뮬레이션]** 외부 변수(기후, 질병)를 반영하여, '현재 가격' 대비 '미래 위험을 반영한 적정 가치'를 예측. |
| **사육/혈통 관리 프로그램** | 개체별 이력 관리, 데이터 기록 | **[정보의 단절성]** 사육 이력이 공공 데이터(예: 기상, 질병)와 연결되지 않아, '정보의 권위'가 개체 사육자에게만 머무름. | **[데이터 통합 허브]** 공신력 있는 공공 데이터와 사육자의 개별 데이터를 통합하여, 국가 차원의 위험 관리 시스템으로 기능. |

## Ⅲ. 핵심 과제 실행을 위한 '데이터 공신력 확보 전략' (Actionable Plan)

'위험 지수 예측 시스템'이 성공하려면, 사용자가 이 데이터가 **'진짜 신뢰할 수 있는 공신력 있는 출처'**에서 왔다고 믿게 만들어야 합니다. 따라서 데이터 확보는 단순히 API 연동을 넘어, **'신뢰의 구조(Structure of Trust)'**를 설계해야 합니다.

### 1. 공신력 확보 방안 (Authority Source Layer)

| 확보 방안 | 구체적 실행 계획 (PoC 단계) | 기대 효과 |
| :--- | :--- | :--- |
| **정부/학술기관 MOU 체결** | **[최우선]** 축산물품질평가원, 농림축산검역본부 등 핵심 기관과의 **데이터 제공 협약(MOU)** 추진. (데이터 활용 목적 명시) | 백프로 서비스 전체에 국가 공신력 뱃지 부착 가능. 가장 강력한 마케팅 무기. |
| **공공 API 데이터 연동 심화** | 기상청(기후), 질병관리청(질병), 농림축산식품부(정책/사료가격) 등 **3대 공공 API**를 핵심 지표로 확정하고, 데이터 변환 로직(Normalization)을 먼저 개발. | 데이터의 최신성, 객관성 확보. (기술적 난이도 높음) |
| **전문가/기관 검증 시스템** | 시스템에 통합된 '위험 지수'에 대해, **최초 계산 후 '검증 완료' 상태**를 부여하고, 검증 주체(예: 백프로-협력 연구원)를 명시. | 데이터의 신뢰도를 높이고, 서비스의 '컨설팅 허브' 이미지를 강화. |
