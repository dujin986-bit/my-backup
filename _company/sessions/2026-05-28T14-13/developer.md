## 📝 1. 핵심 사용자 스토리 (User Story) 정의

사용자의 관점에서, 우리가 해결해야 할 핵심 Pain Point를 스토리로 정의합니다.

| 사용자 유형 | 스토리 ID | 사용자 스토리 (As a [User], I want to [Goal], so that [Benefit]) | 핵심 해결 가치 |
| :--- | :--- | :--- | :--- |
| **축산인 (판매자)** | US-001 | 저는 사육 단계별로 최신 규제 변화와 적정 판매 시점을 알고 싶습니다. $\rightarrow$ 법적 리스크를 줄이고 최대 수익을 얻을 수 있습니다. | **[권위/규제]** 정보의 신뢰성 및 시의성 확보 |
| **축산인 (구매자)** | US-002 | 저는 단순히 시세를 아는 것을 넘어, 이 품종을 사육하는 가장 과학적인 방법론과 전문 지식을 얻고 싶습니다. $\rightarrow$ 사육 과정의 실패 확률을 줄이고 가치를 높일 수 있습니다. | **[지식/전문성]** 공신력 있는 지식 기반 제공 |
| **축산인 (전체)** | US-003 | 저는 제가 가진 사육 노하우를 체계적으로 정리하고 다른 축산인들과 공유하며 인정받고 싶습니다. $\rightarrow$ 커뮤니티 참여를 통해 학습하고 전문가로 성장할 수 있습니다. | **[커뮤니티/성장]** 지식의 자산화 및 교류 |

## ✨ 2. 필수 기능 목록 (MVP Must-Have Features)

MVP는 반드시 작동하는 최소한의 기능을 의미합니다. 위의 스토리 기반으로, 가장 핵심적인 3가지 모듈을 구축합니다.

### A. 🛡️ Authority & Knowledge Hub (핵심 차별화 모듈)
*   **규제 알림 시스템 (Regulatory Alert):** 사육 품목/단계별로 필수적인 법적 변경 사항(예: 백신 접종 주기 변경, 시설 기준 강화)을 자동으로 푸시 알림하고, **공식 출처 링크**를 명시합니다. (가장 중요)
*   **전문가 Q&A/지식 DB:** 정부/학술 자료를 요약하고, 축산인이 질문을 올리면 **검증된 답변(Expert Answer)**이 달리는 구조입니다. (단순 게시판 X)
*   **품종/사육 가이드:** 품종별, 단계별(육성기, 비육기 등) 최적의 사육 가이드라인을 시각화하여 제공합니다.

### B. 📈 Marketplace (최소한의 거래 기능)
*   **실시간 시세 조회:** 기본적인 매물 등록 및 시세 조회 기능은 유지합니다. (키우소의 핵심 기능)
*   **매물 필터링:** 품목, 지역, 사육 단계(MVP 단계에서는 단순하게 '판매 가능'으로 제한) 필터링을 지원합니다.
*   *(MVP 제외 기능)*: 복잡한 거래 시스템(결제 연동, 배송 추적 등)은 V1.0으로 미룹니다.

### C. 👤 Profile & Management
*   **개인화된 대시보드:** 사용자가 자신의 사육 현황(가상의 데이터)과 관련 알림(Regulatory Alert)을 한눈에 볼 수 있게 합니다.
*   **마이 페이지:** 나의 글, 내가 저장한 가이드, 나의 사육 목표 등을 관리합니다.

## 💻 3. 기술 스택 및 아키텍처 초기 제안

MVP 개발 속도와 확장성을 최우선으로 고려했습니다.

### ⚙️ 기술 스택 (Tech Stack)
*   **프론트엔드 (Client):** **React Native (Expo)**
    *   *이유:* iOS, Android, Web까지 단일 코드베이스로 커버 가능하여 MVP 개발 속도가 극대화됩니다.
*   **백엔드 (API Server):** **Python (FastAPI)**
    *   *이유:* 저희의 핵심 가치(Knowledge Hub)는 데이터 처리와 AI 기반의 정보 제공이 필수입니다. Python은 데이터 처리 라이브러리가 강력하고, 향후 AI/ML 기반의 정보 제공 기능(예: "이런 상황엔 어떤 규정을 참고해야 하나요?")을 추가하기에 가장 유리합니다.
*   **데이터베이스:** PostgreSQL (안정적이고 복잡한 관계형 데이터 처리에 강함)

### 시스템 아키텍처 (High Level)

1. **Client Layer:** React Native (모바일 앱)
2. **API Gateway:** FastAPI (RESTful API 제공)
3. **Service Layer:** Python Business Logic (사용자 인증, 데이터 가공, 비즈니스 로직 수행)
4. **Data Layer:** PostgreSQL (데이터 저장소)
5. **Intelligence Layer:** (초기에는 미사용, 추후 확장) LLM 연동 모듈 (지식 기반 검색 및 질문 답변)

### 💡 기술적 제언 (Development Focus)

* **최우선 과제:** 사용자 인증 및 권한 관리 시스템 구축.
* **핵심 구현:** '규정/지식 검색 엔진' (RAG Architecture 기반)을 가장 먼저 설계해야 합니다. 이것이 바로 경쟁 우위가 될 지점입니다.

**요약 결론:**
현재 가장 중요한 것은 '기술 스택'을 정하는 것보다, **'어떤 정보를 어떻게 가공하여 사용자에게 전달할지'**에 대한 구체적인 콘텐츠 전략(Content Strategy)과 정보 구조(Information Architecture)를 먼저 확정하는 것입니다. 이 구조를 바탕으로 Python 백엔드와 React Native 프론트엔드를 구축하는 것이 가장 효율적입니다.
