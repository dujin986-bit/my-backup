## 🚀 금주 주요 개발 계획 (Action Items)

금주 목표는 **'추상적인 계획'**을 **'실행 가능한 시스템 구조'**로 전환하는 것입니다. 각 전문가가 제시한 관점(기술 아키텍처, 비즈니스 로직, 사용자 경험)을 통합하여, 시스템의 뼈대(Skeleton)를 완성하는 데 집중합니다.

---

### 🎯 핵심 개발 목표: 지능형 대시보드 프로토타입 구축

단순한 데이터 표시를 넘어, **'경고(Alert)'** 기능과 **'원인 분석(Root Cause)'** 가이드라인을 제공하는 대시보드 프로토타입을 완성합니다.

### 🛠️ 기술 및 구조적 액션 플랜 (Technical & Architecture)

| 순서 | 액션 항목 | 목표 및 산출물 | 담당 관점 통합 |
| :--- | :--- | :--- | :--- |
| **1** | **API 및 데이터 모델 정의 (Schema Definition)** | 핵심 KPI (사용자 수, 전환율, 지연 시간 등)를 기준으로, 데이터 수집 및 처리를 위한 최종 데이터 모델(Schema)을 정의합니다. (Ex: `user_activity_log` 테이블 구조 확정) | ⚙️ **기술 아키텍처** (데이터 수집 경로 확정) |
| **2** | **핵심 비즈니스 로직 모듈 개발 (Threshold Logic)** | **'경고 임계치(Threshold)'**를 계산하고, 해당 임계치를 벗어날 경우 **'경고 레벨(Level 1, 2, 3)'**을 산출하는 핵심 백엔드 로직을 개발합니다. (Ex: 전환율이 24시간 평균 대비 20% 하락 시 '경고 레벨 2' 부여) | 🧠 **비즈니스 로직** (경고 조건 및 우선순위 확정) |
| **3** | **데이터 시각화 컴포넌트 구현 (Visualization Component)** | 경고 레벨과 추이를 시각적으로 보여주는 차트(Line Chart, Gauge) 컴포넌트를 프론트엔드에서 구현하고, 가짜 데이터(Mock Data)로 연결 테스트를 진행합니다. | ✨ **UX/UI** (데이터를 직관적으로 해석 가능하게 구현) |

---

### 📝 주요 산출물 및 검토 사항 (Deliverables & Checkpoints)

1. **[필수] 데이터 흐름 다이어그램 최종 확정:** (데이터 수집 → 전처리 → 임계치 계산 → DB 저장 → API 제공)의 모든 단계가 명시된 최종 아키텍처 다이어그램을 확정합니다.
2. **[필수] 경고 규칙(Rule Set) 문서화:** 어떤 조건에서 어떤 경고가 발생하는지에 대한 **'Rulebook'**을 문서화하고, 이 규칙을 코드로 구현하는 것을 검토합니다.
3. **[검토] 데이터 소스 연결 테스트:** Mock Data가 아닌, 실제 테스트 데이터 소스(Staging DB)와 API 연결 테스트를 진행하여 데이터 수집의 안정성을 검증합니다.
