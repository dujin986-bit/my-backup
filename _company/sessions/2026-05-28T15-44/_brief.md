# 📋 작업 브리프

**원 명령:** 프로토 타입이 준비되었으면 실행파일을 만들어보고 테스트 해보고, 문제점을 하나씩 잡아나아가자

## 요약
프로토타입(시연용 화면/설계)이 완성되었으므로, 실제 동작하는 실행 파일(Working Prototype)을 제작하고, 이를 테스트하여 발견된 문제점들을 체계적으로 개선하는 단계로 전환해야 합니다. 이 단계는 단순한 코딩을 넘어, 실제 비즈니스 가치 검증에 초점을 맞춰야 합니다.

## 분배
- **💻 코다리**: 최종 위험 지수 예측 시스템의 프로토타입을 실제 작동하는 '시연용 웹 애플리케이션(Demo Web App)' 형태로 구현해 주세요. 이 웹 앱은 Mock API를 사용하여 데이터 연동의 흐름과 핵심 로직(점수 계산, 행동 권고)을 보여주는 것이 목표입니다. 특히, 백엔드 아키텍처 설계(API Endpoint 정의, 데이터 흐름도)를 기반으로, 프론트엔드와 백엔드가 어떻게 통신할지 구체적인 코드 구조(예: React/Vue + Mock JSON API)를 작성하고, 필수적인 에러 처리 및 로딩 상태까지 구현해야 합니다.
- **🎨 Designer**: 개발된 시연용 웹 앱의 UI/UX를 최종 검토하고, 테스트 과정에서 발생할 수 있는 '사용자 혼란'을 최소화하는 방향으로 개선점을 제시해 주세요. 특히, 데이터가 실시간으로 변동할 때의 애니메이션 효과, 데이터 신뢰도를 높이는 출처 표기(Source Citation)의 시각적 강조, 그리고 '행동 매뉴얼' 섹션의 가독성을 최우선으로 점검하여 수정 가능한 디자인 브리프를 작성해 주세요.
- **📱 영숙**: 이 '실행 파일 제작 및 테스트' 단계를 프로젝트 로드맵의 다음 핵심 단계로 정의하고, 필요한 산출물(예: 개발 스펙 문서, 테스트 케이스 목록, 사용자 피드백 수집 계획)을 정리하여 통합 스케줄과 다음 주차의 우선순위 과제 목록을 업데이트해 주세요.
